# 常用工具模块 - 集成测试

## 概述

本目录包含常用工具模块的端到端集成测试，覆盖从工具卡片页到具体工具使用的完整用户流程。

## 测试文件

### `common-tools-flow.test.ts`

端到端集成测试，覆盖以下用户故事：

#### 用户故事1：浏览工具
- 导航到常用工具页面
- 验证工具按分类展示
- 验证工具卡片显示名称、描述和图标
- 验证只显示有工具的分类

#### 用户故事2：使用内置工具（Markdown编辑器）
- 从卡片页点击进入Markdown编辑器
- 验证编辑器和预览面板正确渲染
- 验证默认欢迎内容
- 验证返回按钮功能

#### 用户故事3：使用HTML工具
- 从卡片页点击进入HTML工具运行器
- 验证iframe沙箱安全运行
- 验证HTML地址正确加载
- 验证全屏和返回功能

#### 完整流程测试
- 浏览工具卡片页
- 点击工具卡片进入工具
- 使用工具
- 返回工具卡片页

#### 错误处理
- API加载失败
- 工具不存在
- 工具类型错误

## 运行测试

### 运行所有测试
```bash
npm test
```

### 只运行集成测试
```bash
npm test -- tests/integration
```

### 运行特定测试文件
```bash
npm test -- tests/integration/common-tools-flow.test.ts
```

### 以UI模式运行测试
```bash
npm run test:ui
```

### 生成测试覆盖率报告
```bash
npm run test:coverage
```

## 测试覆盖范围

- ✅ 工具卡片页浏览
- ✅ 工具分类展示
- ✅ Markdown编辑器完整流程
- ✅ HTML工具运行器完整流程
- ✅ 路由导航（卡片页 ↔ 工具页）
- ✅ iframe沙箱安全性
- ✅ 全屏功能
- ✅ 错误处理和重试

## 测试原则

1. **真实用户流程**：测试模拟真实用户的操作路径
2. **端到端覆盖**：从页面导航到功能使用的完整流程
3. **关键路径优先**：重点测试核心用户故事
4. **错误处理**：验证各种错误场景的处理

## 注意事项

- 集成测试使用Mock API，不会调用真实后端
- 测试使用jsdom环境模拟浏览器
- 全屏功能在测试环境中可能受限，主要验证调用逻辑
- iframe内容加载在测试环境中模拟，不会真正加载HTML文件
