# 党建CRUD功能TDD完善方案

**日期：** 2026-03-05
**目标：** 通过TDD确保党员管理、组织生活、党费管理三个模块的CRUD功能完善可用

## 问题概述

当前三个模块存在以下问题：
- 列表查询：分页、筛选、搜索功能不正常
- 数据添加：验证规则、必填字段、默认值处理有问题
- 数据编辑：更新逻辑、字段覆盖、数据一致性有问题
- 数据库适配：实际表结构与ORM模型定义不匹配

## 解决方案：逐模块完整TDD

### 实施顺序
1. 党员管理 (已有部分测试，优先完善)
2. 组织生活 (参考党员模式)
3. 党费管理 (从零开始)

### 每个模块的测试清单

```
✅ 列表查询
  - 空列表返回200
  - 正常列表带分页
  - 分页参数验证
  - 筛选功能验证

✅ 详情查询
  - 获取存在记录成功
  - 获取不存在记录返回404

✅ 创建记录
  - 提供完整数据创建成功(201)
  - 只提供必填字段创建成功
  - 缺少必填字段返回422
  - 数据格式错误返回422

✅ 更新记录
  - 部分更新成功(200)
  - 更新不存在记录返回404

✅ 删除记录
  - 删除成功返回204
  - 删除不存在记录返回404
```

### 测试文件结构

```
backend/tests/integration/routers/
├── test_party_members.py      # 党员管理测试
├── test_organization_lives.py # 组织生活测试

```

### 技术要点

1. **只写集成测试** - 测试API端点，覆盖实际使用场景
2. **发现一个修复一个** - 测试失败立即修复代码，直到全绿
3. **统一错误处理** - 422(验证错误)、400(业务错误)、404(不存在)
4. **数据库隔离** - 使用pytest fixture确保测试独立
